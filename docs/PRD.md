# 專案名稱：食譜收藏夾系統

## 1. 專案概述

### 背景與動機
現代人常在各大網站、社群平台看到想嘗試的食譜，但缺乏一個統整的平台來收藏與管理。為了解決食譜散落各處、難以搜尋以及食材準備不便的問題，我們決定開發「食譜收藏夾系統」，讓使用者能輕鬆收藏、搜尋且有效率的準備烹飪材料。

### 目標用戶
- **一般用戶**：喜愛下廚、尋找新菜色，需要管理個人日常食譜與食材採買清單的人。
- **管理員**：負責管理系統、維護食譜分類與處理異常內容的人員。

### 核心價值主張
提供一個一站式的食譜管理服務，從靈感收藏、精準搜尋（食材組合搜尋）到採買準備（一鍵生成購買清單、分量換算），無縫串聯下廚的每一個環節，提升在家烹飪的意願與效率。

---

## 2. 功能需求

### 1. 儲存食譜
- **一般用戶**：作為一般用戶，我希望能夠新增並儲存喜歡的食譜細節（包含名稱、食材、步驟、圖片等），以便我日後隨時查看。

### 2. 瀏覽、搜尋食譜
- **一般用戶**：作為一般用戶，我希望能夠透過關鍵字或分類來瀏覽與搜尋食譜，以便快速找到我想做的菜。

### 3. 從食材組合搜尋食譜
- **一般用戶**：作為一般用戶，我希望能夠輸入家裡現有的多種食材來搜尋可做的食譜，以便我能清空冰箱庫存且不用煩惱要煮什麼。

### 4. 一鍵生成購買清單
- **一般用戶**：作為一般用戶，我希望能從一份或多份想做的食譜中自動擷取並加總所需食材，生成購物清單，以便我去超市採買時不會遺漏。

### 5. 分量自動換算功能
- **一般用戶**：作為一般用戶，我希望能在食譜頁面自由調整想烹飪的份量（如兩人份改為四人份），讓系統自動換算食材用量，以便我能精準準備食材。

---

## 3. 非功能需求

### 技術限制
- 本專案指定使用 **Flask** 作為後端框架。
- 採用 **Jinja2** 進行前端模板渲染。
- 資料庫使用輕量級的關聯式資料庫 **SQLite** 進行儲存。

### 效能與安全考量
- **效能**：食譜圖片需考量檔案大小限制，並在前端做進階的快取處理以提升載入速度；搜尋功能應確保在資料量達到千筆時，回應時間仍小於 1 秒。
- **安全**：需對使用者輸入進行驗證與過濾，防止 XSS（跨站腳本攻擊）與 SQL Injection（隱碼攻擊）；涉及管理員權限的操作必須嚴格驗證。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必須具備)
- 使用者註冊與登入功能
- 新增、編輯、刪除個人儲存的食譜
- 瀏覽與關鍵字搜尋食譜
- 從食材組合搜尋食譜的基本功能
- 基礎的食譜份量自動換算功能

### Should Have (建議具備)
- 一鍵生成並管理食材購買清單
- 系統首頁的最新食譜推薦
- 食譜標籤系統（例如：素食、快速上菜）

### Nice to Have (有會更好)
- 社群分享功能（將食譜分享給其他使用者）
- 上傳自訂食譜圖片
- 管理員後台頁面與內容管理審核機制
